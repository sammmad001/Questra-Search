# Questra-Search → 个人知识库 集成部署步骤

## 前置条件

1. 个人知识库（KB）项目已启动并可访问（默认 http://localhost:8080）
2. KB 侧 `KNOWLEDGE_API_TOKEN` 已配置（查看 `/kb/.env`）
3. Questra-Search 项目依赖已安装（`httpx` 通常已存在）

## 部署步骤

### 第1步：复制文件

将 `questra-search-kb-integration/` 目录下的文件覆盖到 Questra-Search 项目：

```bash
# 假设 Questra-Search 项目位于 /Users/sam/Desktop/Questra-Search
cp -r questra-search-kb-integration/app/* /Users/sam/Desktop/Questra-Search/app/
cp questra-search-kb-integration/.env.example /Users/sam/Desktop/Questra-Search/.env.example
```

### 第2步：配置环境变量

编辑 Questra-Search 的 `.env` 文件，新增以下配置：

```bash
# ── 个人知识库集成 ──
KB_API_BASE=http://localhost:8080
KB_API_TOKEN=<与KB侧 KNOWLEDGE_API_TOKEN 一致的值>
KB_AUTO_INGEST=true
```

### 第3步：数据库迁移

**方式A（自动）**：重启 Questra-Search 服务，`init_db()` 中的 `_MIGRATIONS` 会自动执行 `ALTER TABLE messages ADD COLUMN kb_sent`。

**方式B（手动）**：如果自动迁移因 SQLite 版本问题失败：
```bash
sqlite3 data/questra_search.db "ALTER TABLE messages ADD COLUMN kb_sent INTEGER NOT NULL DEFAULT 0;"
```

### 第4步：重启服务

```bash
cd /Users/sam/Desktop/Questra-Search
python server.py
```

启动日志中应看到：
```
KB 重试调度器已启动（间隔=300s）
```

### 第5步：验证

1. 发送一条聊天消息，等待完成
2. 查看 Questra-Search 日志，确认看到 `KB 入库成功: session=X tokens=Y`
3. 检查 KB 侧状态 API：
```bash
curl -H "Authorization: Bearer <TOKEN>" http://localhost:8080/api/ingest/status
```

## 修改文件清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `app/config.py` | 修改 | 新增 KB_API_BASE、KB_API_TOKEN、KB_AUTO_INGEST |
| `app/database.py` | 修改 | messages 表新增 kb_sent 列 + 索引 + 迁移语句 |
| `app/main.py` | 修改 | lifespan 中启动/停止 KbRetrySender |
| `app/routers/chat.py` | 修改 | 查询 session title/model 传递给 StreamRecorder |
| `app/services/stream_recorder.py` | 修改 | 新增 _post_to_kb() fire-and-forget hook |
| `app/services/kb_retry.py` | **新增** | 定时扫描重发未成功消息 |
| `.env.example` | 修改 | 新增 KB_* 配置项 |

## 回滚方案

如需回滚，将以上修改的文件还原为备份即可，数据库中的 `kb_sent` 列保留不影响原有功能。
